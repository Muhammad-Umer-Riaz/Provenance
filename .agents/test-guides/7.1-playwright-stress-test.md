## Part 1 — What changed (Modules 7 and 7.1)

### Module 7 (eval framework — already committed, not under test here)
- `eval/field_accuracy.py` — runs orchestrator on 15 synthetic test cases; exact-matches
  7 deterministic fields; LLM judge with groundedness threshold gate
- `eval/time_to_approval.py` — audit_log regen/edit analysis for a real report UUID
- `eval/compare_runs.py` — diffs two eval result JSONs for prompt regression
- `eval/test_sets/sqr_cases.json` — 15 synthetic SQR intake fixtures
- `frontend/src/pages/ReportReview.tsx` — failed-status fields now show the Edit input
  (backend already allowed value edits on failed fields; UI was blocking it)

### Module 7.1 (UI overhaul — under test in this session)

**Backend:**
- `templates/supplier-qualification-report.yaml` — all 6 validation rules rewritten
  in valid simpleeval Python. They were previously broken (used a custom DSL that
  simpleeval could not parse, so all 6 always fired as warnings regardless of data).
  Fixed rules:
  - `score_weights_sum` → `abs(sum([r['weight'] for r in audit_scores]) - 1.0) < 0.001`
  - `verdict_car_consistency` → `qualification_verdict == 'Preferred' or len(corrective_actions) >= 1`
  - `overdue_car_date_check` → `all([car['due_date'] >= today for car in corrective_actions if car['status'] in ['Open', 'In progress']])`
  - `high_tier_risk_count` → `overall_risk_tier != 'High' or len(risk_register) >= 3`
  - `high_risk_car_coverage` → `not any([r['likelihood'] * r['impact'] >= 15 for r in risk_register]) or len(corrective_actions) >= 1`
  - `performance_verdict_alignment` → `not (breach_count >= 2 and qualification_verdict == 'Preferred')`
- `backend/app/validator.py` — injects `today` (ISO date string) into evaluator context

**Frontend:**
- `frontend/src/pages/GeneratePage.tsx` — step ribbon added: `✓ Intake → Generate (active) → Review`
- `frontend/src/pages/ReportReview.tsx`:
  - Step ribbon: `✓ Intake → ✓ Generate → Review (active)`
  - Center pane now has two tabs: **Report** (default) and **Validation**
  - **Validation tab**: shows issues (passed=false) first with amber/red severity styling,
    then a collapsible "Show N passed checks" section. Badge on tab shows issue count.
  - Left pane: warnings block removed (moved to Validation tab)
  - Section headers: `§` prefix removed — now plain `Supplier Header`, `Audit Scorecard`, etc.

## Part 8 — If something breaks

Common failure modes and where to look:

| Symptom | Where to look |
|---|---|
| Validation tab always shows 6 issues | Check `validator.py` — `ctx` must include `today`; check YAML rules are saved correctly |
| Step ribbon not showing | Check `ReportReview.tsx` and `GeneratePage.tsx` import of `ChevronRight` from lucide-react |
| Section headers still show `§` | Check `ReportReview.tsx` — search for `§` |
| Export button never enables | Check `nonFailedFields` logic — failed fields should be excluded from the gate |
| Edit button missing on failed field | Check `canEdit` logic in `FieldPanel` — `'failed'` should NOT be in the `['pending', 'generating']` exclusion list |
| Backend 500 on validation | Check `validator.py` — `ctx` dict must be created with `{**context, "today": ...}` |
