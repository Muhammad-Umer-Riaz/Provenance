# Module 7.1: Review UI Overhaul ŌĆö Plan

## Context

Four problems surfaced after Module 6 shipped:
1. All 6 validation warnings always fired on every report ŌĆö the YAML rules used a custom
   DSL (wildcards, `for each`, uppercase `OR/AND/NOT`, undefined `sla_breach_flags`) that
   simpleeval cannot parse. Every rule threw an exception ŌåÆ `passed=False`. Meaningless noise.
2. Warnings occupied permanent left-pane space even on clean reports.
3. No visual indication of workflow position (Intake ŌåÆ Generate ŌåÆ Review) on either
   the Generate or Review pages.
4. Section headers in the center pane used `┬¦` as a decorative prefix adding no value.

## Files modified

| File | Change |
|---|---|
| `templates/supplier-qualification-report.yaml` | Rewrote all 6 validation rule `check` expressions in valid simpleeval Python |
| `backend/app/validator.py` | Injected `today` (ISO string) into evaluator context for overdue CAR date check |
| `frontend/src/pages/ReportReview.tsx` | Step ribbon + Report/Validation tabs + removed warnings from left pane + removed ┬¦ |
| `frontend/src/pages/GeneratePage.tsx` | Step ribbon (Intake=done, Generate=active, Review=upcoming) |

## Validation rule fixes

| Rule ID | Problem | Fix |
|---|---|---|
| `score_weights_sum` | `[*]` wildcard not valid Python | `abs(sum([r['weight'] for r in audit_scores]) - 1.0) < 0.001` |
| `verdict_car_consistency` | Uppercase `OR` | `qualification_verdict == 'Preferred' or len(corrective_actions) >= 1` |
| `overdue_car_date_check` | `for each` syntax | `all([car['due_date'] >= today for car in corrective_actions if car['status'] in ['Open', 'In progress']])` |
| `high_tier_risk_count` | Uppercase `OR` | `overall_risk_tier != 'High' or len(risk_register) >= 3` |
| `high_risk_car_coverage` | `for each`/`exists` syntax | `not any([r['likelihood'] * r['impact'] >= 15 for r in risk_register]) or len(corrective_actions) >= 1` |
| `performance_verdict_alignment` | Uppercase `NOT`/`AND`, undefined `sla_breach_flags` | `not (breach_count >= 2 and qualification_verdict == 'Preferred')` |

## UI changes

- **Step ribbon**: thin `bg-muted/30` strip above the header on both GeneratePage and
  ReportReview. Shows Intake ŌåÆ Generate ŌåÆ Review with Ō£ō ticks on completed steps.
- **Validation tab**: center pane now has Report / Validation tabs. Validation tab shows
  issues (passed=false) grouped first with amber/red styling by severity, then a
  collapsible "Show N passed checks" section in green. Badge on tab shows issue count.
- **Left pane**: warnings block removed entirely ŌĆö left pane now shows only score/verdict
  card and intake summary.
- **Section headers**: `┬¦ {formatId(sectionId)}` ŌåÆ `{formatId(sectionId)}`.
