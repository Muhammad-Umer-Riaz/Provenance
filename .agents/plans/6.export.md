# Module 6: Export — Implementation Plan

## Context

Module 5 left the review UI with an export button stub that navigated to a non-existent
`/reports/:id/export` route. Module 6 replaces that stub with a working format dropdown
that triggers direct synchronous file downloads — PDF, DOCX, and JSON — without leaving
the review page.

**Note:** `docs/STATUS.md` lists WeasyPrint as the PDF renderer. This is outdated. Decision 14
in `docs/DECISIONS.md` documents Playwright/Chromium as the correct choice. The plan follows
the decision document.

**Confirmed design choices:**
- Synchronous export: POST /export waits and streams the file back (no background task)
- Direct download: FastAPI `Response` returns file bytes (no Supabase Storage)
- Export UI: Format dropdown on the existing export button in `ReportReview.tsx` — no new route
- JSON scope: Full provenance — fields + strategies + approval status + intake_data +
  validation_warnings + full audit_log rows for the report
- DOCX fidelity: Structured document — section headings, formatted tables, bold labels

### B2 — `backend/app/routes/export.py`

```
POST /api/reports/{report_id}/export?format=pdf|docx|json
```

Route logic (in order):
1. `_verify_report_ownership(report_id, user_id)` — reuse from `fields.py`
2. Fetch all fields ordered by `field_index`:
   `supabase.table("report_fields").select("*").eq("report_id", report_id).order("field_index").execute()`
3. **Approval gate** (Decision 6 — backend-enforced):
   If any field has `status not in ('approved', 'failed')` → raise `HTTPException(409)` with
   `detail` listing the count and field IDs that are not yet approved.
4. Route to renderer:
   - `pdf` → `await generate_pdf(report, fields)`
   - `docx` → `generate_docx(report, fields)`
   - `json` → `generate_json(report, fields, audit_trail)`
5. Log to `audit_log`: `event_type="exported"`, `field_id="_export"`, 
   `inputs_snapshot={"format": format}`, `output_value=None`, `model=None`
6. Return `Response(content=bytes, media_type=..., headers={"Content-Disposition": "attachment; filename=..."})`

Content-types:
- PDF: `application/pdf`, filename `report-{id}.pdf`
- DOCX: `application/vnd.openxmlformats-officedocument.wordprocessingml.document`, filename `report-{id}.docx`
- JSON: `application/json`, filename `report-{id}.json`

For JSON, the route must first fetch audit_trail rows:
`supabase.table("audit_log").select("*").eq("report_id", report_id).order("created_at").execute()`

Reuse helpers from `fields.py`:
- `_verify_report_ownership` (import directly)
- `get_current_user_id` (import from `reports.py`)

### B3 — `backend/app/services/pdf_renderer.py`

`async def generate_pdf(report: dict, fields: list[dict]) -> bytes`

**HTML generation** (pure Python, no Jinja2):
- Build one HTML string with inline CSS
- Print CSS: A4 format, 20mm margins, `page-break-inside: avoid` per section
- Header block: supplier name (large), verdict badge (colour-coded by tier:
  Preferred=green, Conditional=amber, Probationary=orange, Rejected=red), score, evaluator, date
- Group fields by `section_id`, preserving `field_index` order
- For each section: `<h2>§ {section_id}</h2>` + field rows
- Field row structure:
  - `<div class="field-label">{field_id}</div>`
  - Value rendering (detect by content):
    - Starts with `[` → parse JSON → render as HTML `<table>`
    - Starts with `{` → parse JSON → render as 2-column `<table>`
    - Otherwise → `<div class="field-value">{value}</div>`
  - Small `<span class="strategy-tag">{strategy}</span>` in muted text

**Playwright call:**
```python
from playwright.async_api import async_playwright

async with async_playwright() as p:
    browser = await p.chromium.launch()
    page = await browser.new_page()
    await page.set_content(html, wait_until="load")
    pdf_bytes = await page.pdf(format="A4", print_background=True, margin={...})
    await browser.close()
return pdf_bytes
```

The export route handler is an `async def`, so `await generate_pdf(...)` works without a thread executor.

### B4 — `backend/app/services/docx_renderer.py`

`def generate_docx(report: dict, fields: list[dict]) -> bytes`

Structure:
1. `doc = Document()`
2. Title: `doc.add_heading("Supplier Qualification Report", 0)`
3. Subtitle paragraph: `{supplier_name} · {verdict} · Score {score} · {date}`
4. Horizontal rule (paragraph with bottom border style)
5. For each section (grouped by `section_id`, preserving `field_index` order):
   - `doc.add_heading(format_section_name(section_id), level=1)`
   - For each field:
     - Parse value: try `json.loads(value)` → if list → Word table; if dict → 2-col table;
       else → labelled paragraph
     - **List value (table data):**
       - `table = doc.add_table(rows=1+len(data), cols=len(data[0]))`
       - First row: column headers (bold); remaining rows: data cells
       - `table.style = "Table Grid"`
     - **Dict value:**
       - 2-column table: key (bold) | value
     - **String value:**
       - `p = doc.add_paragraph(); run = p.add_run(f"{label}: "); run.bold = True; p.add_run(str(value))`
6. Save to `io.BytesIO`, return `.getvalue()`

Helper `format_section_name(section_id: str) -> str`: replaces underscores with spaces, title-cases.

### F1 — `frontend/src/lib/api.ts` — `exportReport()`

Cannot reuse `apiFetch` (which calls `res.json()`) — blob download requires `res.blob()`.

```typescript
export async function exportReport(
  reportId: string,
  format: 'pdf' | 'docx' | 'json',
): Promise<void> {
  const { data: { session } } = await supabase.auth.getSession()
  const res = await fetch(
    `${BASE}/api/reports/${reportId}/export?format=${format}`,
    { method: 'POST', headers: { Authorization: `Bearer ${session?.access_token ?? ''}` } },
  )
  if (!res.ok) {
    const body = await res.text()
    throw new Error(`Export failed (${res.status}): ${body}`)
  }
  const blob = await res.blob()
  const url = URL.createObjectURL(blob)
  const a = document.createElement('a')
  a.href = url
  a.download = `report-${reportId}.${format}`
  document.body.appendChild(a)
  a.click()
  document.body.removeChild(a)
  URL.revokeObjectURL(url)
}
```

## Approval gate note (field_id in audit_log)

`audit_log` rows require `field_id`. Export is a report-level event with no field.
Use sentinel value `"_export"` as `field_id` — clearly non-field, not a UUID, does not
collide with any real field_id. The audit_log query in the JSON renderer should exclude
rows where `field_id = '_export'` from the `audit_trail` array (those are export meta-events,
not field generation events).
