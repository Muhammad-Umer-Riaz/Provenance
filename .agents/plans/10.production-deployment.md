# Module 10: Production Deployment

> **Status: Deployed — 18 May 2026**
> See `docs/STATUS.md § Module 10` for the as-deployed record, bug fixes, and actual resource IDs.
>
> **Key deviations from this plan:**
> - Port is **8080** (not 8000) — `start.sh` had a bug; `frontend/.env` confirmed 8080 is the real port
> - Ubuntu **26.04** used (not 22.04 — 22.04 was not in the EC2 Quick Start list)
> - CloudFront new wizard UI differs: has a "Get started" name step first; raw IP addresses rejected as origin — use EC2 public DNS hostname (`ec2-13-62-248-243.eu-north-1.compute.amazonaws.com`)
> - `VITE_BACKEND_URL` env var must be set in the shell (`$env:VITE_BACKEND_URL = "..."`) before `npx vite build` — `.env.production` file alone was not picked up reliably in PowerShell

## Context

This plan deploys Provenance (ReportForge) to AWS for a ~1-month demo window.
No custom domain. HTTPS is handled by CloudFront's default `*.cloudfront.net` certificate.
Budget: ~$1–2/month from $86 AWS credits. Region: eu-north-1 (Stockholm).
AWS account is within its first 12 months → EC2 free tier applies → using t3.micro (free).
Swap space (1GB) is added to the instance to prevent OOM during Playwright PDF exports.

**Architecture:**
```
Browser ──HTTPS──► CloudFront 1 (*.cloudfront.net) ──► S3 bucket (React build)
Browser ──HTTPS──► CloudFront 2 (*.cloudfront.net) ──► EC2 Nginx :80 ──► uvicorn :8080
```

CloudFront 2 in front of EC2 solves the mixed-content problem (HTTPS frontend calling HTTP backend) without requiring a domain or Certbot.

**Why this order matters:**
- CloudFront 2 (backend) must be created BEFORE the frontend is built, because `VITE_BACKEND_URL` is baked into the JS bundle at build time.
- CloudFront 1 (frontend) URL is needed for the backend's `FRONTEND_URL` (CORS origin).

### 1.5 — GitHub Actions workflow

**File: `.github/workflows/deploy.yml`** (new)

```yaml
name: Deploy

on:
  push:
    branches: [main]

env:
  AWS_REGION: eu-north-1
  ECR_REPOSITORY: provenance-backend

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: eu-north-1

      - name: Login to ECR
        id: login-ecr
        uses: aws-actions/amazon-ecr-login@v2

      - name: Build and push backend image
        env:
          ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
          IMAGE_TAG: ${{ github.sha }}
        run: |
          docker build -f backend/Dockerfile -t $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG .
          docker tag $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG $ECR_REGISTRY/$ECR_REPOSITORY:latest
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:latest

      - name: Deploy backend to EC2
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.EC2_HOST }}
          username: ubuntu
          key: ${{ secrets.EC2_SSH_KEY }}
          envs: ECR_REGISTRY
          script: |
            aws ecr get-login-password --region eu-north-1 | \
              docker login --username AWS --password-stdin ${{ secrets.ECR_REGISTRY }}
            export ECR_IMAGE=${{ secrets.ECR_REGISTRY }}/provenance-backend:latest
            docker compose pull
            docker compose up -d
            docker image prune -f

      - name: Build frontend
        working-directory: frontend
        env:
          VITE_BACKEND_URL: ${{ secrets.VITE_BACKEND_URL }}
          VITE_SUPABASE_URL: ${{ secrets.VITE_SUPABASE_URL }}
          VITE_SUPABASE_ANON_KEY: ${{ secrets.VITE_SUPABASE_ANON_KEY }}
        run: |
          echo "VITE_BACKEND_URL=$VITE_BACKEND_URL" > .env
          echo "VITE_SUPABASE_URL=$VITE_SUPABASE_URL" >> .env
          echo "VITE_SUPABASE_ANON_KEY=$VITE_SUPABASE_ANON_KEY" >> .env
          npm ci
          npm run build

      - name: Upload frontend to S3
        run: |
          aws s3 sync frontend/dist/ s3://${{ secrets.S3_BUCKET }} --delete

      - name: Invalidate CloudFront cache
        run: |
          aws cloudfront create-invalidation \
            --distribution-id ${{ secrets.CF_FRONTEND_DIST_ID }} \
            --paths "/*"
```

### Step 2.1 — IAM: Create deployment user

*This user's credentials go into GitHub Actions. It can push to ECR and upload to S3.*

2. Left sidebar → **Users** → click **Create user**
3. **User name:** `provenance-deployer` → click **Next**
4. **Permissions:** select **Attach policies directly**
5. Search and check these policies:
   - `AmazonEC2ContainerRegistryPowerUser`
   - `AmazonS3FullAccess`
   - `CloudFrontFullAccess`
6. Click **Next** → **Create user**
7. Click the user name `provenance-deployer` → **Security credentials** tab
8. Click **Create access key** → choose **Application running outside AWS** → **Next**
9. Click **Create access key**
10. **Download the .csv** or copy both values now — you will NOT see the secret again.

### Step 2.6 — EC2: Install Docker (SSH in)

```bash
# From your terminal — replace <ELASTIC-IP> and path to your .pem
ssh -i provenance-key.pem ubuntu@<ELASTIC-IP>
```

Run these commands on the server:
```bash
# Add 1GB swap — prevents OOM when Playwright launches Chrome during PDF export
sudo fallocate -l 1G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab   # persist across reboots

# Update and install Docker
sudo apt-get update
sudo apt-get install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
echo "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/docker.asc] \
  https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

# Allow ubuntu user to run docker without sudo
sudo usermod -aG docker ubuntu
newgrp docker   # reload group — or log out and back in

# Install AWS CLI
sudo apt-get install -y awscli

# Verify
docker --version
docker compose version
```

### Step 2.7 — EC2: Nginx setup (SSH)

```bash

        proxy_pass         http://localhost:8080;
        proxy_set_header   Host              $host;

        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;

sudo rm /etc/nginx/sites-enabled/default    # remove default placeholder
sudo nginx -t                                # must say "syntax is ok"
sudo systemctl restart nginx
sudo systemctl enable nginx
```

### Step 2.12 — S3: Create frontend bucket

1. Search "S3" → **Create bucket**

**General configuration:**
- **Bucket name:** `provenance-frontend-[your-account-id]` (must be globally unique)
- **AWS Region:** eu-north-1

**Object Ownership:**
- Select **ACLs disabled** (recommended)

**Block Public Access settings:**
- **Uncheck** "Block all public access"
- Confirm the warning checkbox

Click **Create bucket**.

**Enable static website hosting:**
1. Click into the bucket → **Properties** tab
2. Scroll to **Static website hosting** → Edit
3. Enable → **Host a static website**
4. Index document: `index.html`
5. Error document: `index.html` (React Router handles 404s itself)
6. Save

**Bucket policy (public read):**
1. **Permissions** tab → **Bucket policy** → Edit
2. Paste (replace `YOUR-BUCKET-NAME`):
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
    }
  ]
}
```
3. Save
