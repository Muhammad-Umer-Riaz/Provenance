# Module 2: YAML Template Engine + Strategy Handlers

## Context

Module 1 produced a running FastAPI + React scaffold with Supabase schema, auth, and a YAML
template loader that validates top-level keys only. Module 2 builds the entire generation engine
on top of that scaffold: a full Pydantic schema for every field type in the YAML, seven strategy
handlers, an expression evaluator, and an orchestrator that runs them in the correct order.

The SQR YAML template (finalized in the template design session) drives all implementation
decisions. Every feature the engine must support — 26 calculator expressions, 4 lookup modes,
2 classifier paths, 6 narrative_llm fields — is catalogued from that file. Nothing is
speculative; the template is the spec.

After Module 2, a developer can call `ReportOrchestrator(template, intake_data, report_id).generate()`
and receive a fully-populated context dict with every deterministic field resolved and every
narrative paragraph generated. The Supabase write layer and async generation API come in Module 4.

### Step 5 — `strategies/utils.py` [Simple]

Single shared function used by both `lookup.py` and `template_fill.py`:

```python
import re
from typing import Any

def interpolate(text: str, context: dict[str, Any]) -> str:
    """Replace {{token}} and {{a.b}} placeholders with context values."""
    def replacer(m):
        key = m.group(1).strip()
        parts = key.split(".")
        val = context
        for part in parts:
            val = val[part]
        return str(val)
    return re.sub(r'\{\{([^}]+)\}\}', replacer, text)
```

### Step 6 — `strategies/lookup.py` [Simple]

```python
def execute_lookup(source: str, lookup_sources: dict, context: dict) -> str:
    """
    Two source forms:
      "risk_disclosure_clause"                      → plain key
      "qualification_scope_statements[commodity_category]" → dict key lookup

```

Parse the source string with `re.match(r'^(\w+)\[(\w+)\]$', source)`. If match: first group is
the outer key, second group is the context key whose value indexes the inner dict.

After retrieval, call `interpolate(text, context)` from `utils.py`.

### Step 8 — `strategies/calculator.py` [Complex]

This is the hardest file. Build it in isolation and verify each sub-component independently
before composing.

#### 8a. Pre-processor pipeline (apply in this exact order)

| Step | Pattern → Replacement | Purpose |
|---|---|---|
| 1 | `today + N_months` → `__today_plus_months(N)` | Date arithmetic |
| 1b | bare `today` → `__today()` | Current date |
| 2 | `table[*].field` → `__col(table, 'field')` | Star projection |
| 3 | `__col(a,'f') * __col(b,'g')` → `__zip_mul(...)` | Element-wise multiply |
| 4 | `table[N].field` → `table[N]['field']` | Integer index + dot |
| 5 | `sla_thresholds.x.y` → `__get(sla_thresholds, 'x', 'y')` | Nested dict with `pass` keyword |
| 6 | `cond ? then : else` → `(then) if (cond) else (else)` | JS ternary — always last |

#### 8b. JS ternary converter (Step 6 above) — Critical algorithm

Scan the expression string tracking `paren_depth`, `bracket_depth`, `in_single_quote`,
`in_double_quote`. A `?` or `:` only counts when all depths are 0 and not inside a string.

```python
def _find_ternary_splits(expr: str) -> tuple[int | None, int | None]:
    """Return (q_pos, c_pos) — positions of first top-level ? and its matching :"""
    depth = 0; in_sq = False; in_dq = False; q_pos = None
    for i, ch in enumerate(expr):
        if ch == "'" and not in_dq: in_sq = not in_sq
        elif ch == '"' and not in_sq: in_dq = not in_dq
        elif in_sq or in_dq: continue
        elif ch in ('(', '['): depth += 1
        elif ch in (')', ']'): depth -= 1
        elif ch == '?' and depth == 0: q_pos = i; break
    if q_pos is None: return None, None
    depth = 0; in_sq = False; in_dq = False
    for i in range(q_pos + 1, len(expr)):
        ch = expr[i]
        if ch == "'" and not in_dq: in_sq = not in_sq
        elif ch == '"' and not in_sq: in_dq = not in_dq
        elif in_sq or in_dq: continue
        elif ch in ('(', '['): depth += 1
        elif ch in (')', ']'): depth -= 1
        elif ch == ':' and depth == 0: return q_pos, i
    return q_pos, None

def _convert_js_ternary(expr: str) -> str:
    q_pos, c_pos = _find_ternary_splits(expr)
    if q_pos is None or c_pos is None: return expr
    cond = expr[:q_pos].strip()
    then = expr[q_pos + 1:c_pos].strip()
    else_ = _convert_js_ternary(expr[c_pos + 1:].strip())  # recurse for chaining
    return f"({then}) if ({cond}) else ({else_})"
```

Write `test_preprocess_js_ternary` and verify against all 4 multi-level expressions in the
YAML before writing a single line of simpleeval integration.

#### 8c. Custom functions

```python
CUSTOM_FUNCTIONS = {
    "sum": lambda items: builtins.sum(items),
    "min": lambda items: builtins.min(items),
    "max": lambda items: builtins.max(items),
    "len": builtins.len,
    "round": builtins.round,
    "count": _agg_count,       # count(table) or count(table, field='value')
    "min_by": _agg_min_by,     # min_by(table, key='field') → row dict
    "__col": lambda t, f: [r[f] for r in t],
    "__zip_mul": lambda a, b: [x * y for x, y in zip(a, b)],
    "__get": lambda obj, *keys: functools.reduce(lambda o, k: o[k], keys, obj),
    "__today": lambda: datetime.date.today(),
    "__today_plus_months": lambda n: datetime.date.today() + relativedelta(months=n),
}
```

`_agg_count(table, **kwargs)`: if no kwargs, `len(table)`; else filter rows by kwargs then `len`.

`_agg_min_by(table, key)`: `min(table, key=lambda r: r[key])`.

def build_eval_context(context, intake_data, lookup_sources, sla_thresholds) -> dict:

#### 8e. Main entry point

```python
from simpleeval import EvalWithCompoundTypes

def evaluate_expression(expression, context, intake_data, lookup_sources, sla_thresholds) -> Any:
    preprocessed = preprocess(expression.strip())
    names = build_eval_context(context, intake_data, lookup_sources, sla_thresholds)
    ev = EvalWithCompoundTypes(names=names, functions=CUSTOM_FUNCTIONS)
    return ev.eval(preprocessed)
```

Use `EvalWithCompoundTypes` (not `SimpleEval`) — required for dict/list subscript access.

#### 8f. `execute_calculator` public function

```python
def execute_calculator(field: FieldSchema, context, intake_data, lookup_sources, sla_thresholds) -> Any:
    result = evaluate_expression(field.expression, context, intake_data, lookup_sources, sla_thresholds)
    if result is None and field.null_value is not None:
        return field.null_value
    if field.format == "YYYY-MM-DD" and hasattr(result, 'strftime'):
        return result.strftime("%Y-%m-%d")
    if field.format == "0.00" and isinstance(result, (int, float)):
        return round(float(result), 2)
    return result
```

### Step 10 — `strategies/classifier.py` [Medium]

Two execution paths (AD-020):

**Path A — Rule-based** (when `field.rules` is not None):

```python
# 1. Evaluate field.input as an expression to get the input value
input_val = evaluate_expression(field.input, context, intake_data, lookup_sources, sla_thresholds)

# 2. Infer the variable name the rules expect from the first rule's condition
rule_var = re.match(r'^\s*(\w+)', field.rules[0].condition).group(1)

# 3. Inject input_val under that name into a rule evaluation context
rule_context = {**context, rule_var: input_val}

# 4. Evaluate each rule.condition in order; return first match
for rule in field.rules:
    if evaluate_expression(rule.condition, rule_context, intake_data, lookup_sources, sla_thresholds):
        return rule.output

raise ValueError(f"No matching rule for {field.id}, input={input_val}")
```

This handles both SQR classifiers:
- `qualification_verdict`: `field.input = "composite_score"` → `input_val = 3.75`; rules reference
  `composite_score` which is already in `context`, so the injection is redundant but harmless.
- `overall_risk_tier`: `field.input = "max(risk_register[*].priority_score)"` → `input_val = 20`;
  rules reference `max_priority`, so inject as `rule_context["max_priority"] = 20`.

**Path B — LLM fallback** (when `field.rules` is None):

Async call via `AsyncOpenAI` with structured output `ClassifierOutput(label: str, confidence: float)`.
Max 3 retries with exponential backoff. Only used for classifiers with no threshold rules.

### Step 11 — `strategies/narrative_llm.py` [Medium]

```python
async def execute_narrative_llm(field, context, intake_data, lookup_sources, openai_client, model, max_retries=3) -> str:
    # 1. Render prompt: replace {{token}} in field.prompt using interpolate()
    #    For tokens that are expressions (contain parens/brackets), evaluate via evaluate_expression()
    rendered_prompt = _render_prompt(field.prompt, context, intake_data, lookup_sources, sla_thresholds={})

    # 2. Build system prompt (include exemplars if present)
    system = _build_system_prompt(field)

    # 3. Call OpenRouter with retry
    for attempt in range(max_retries):
        try:
            resp = await openai_client.beta.chat.completions.parse(
                model=model,
                messages=[{"role": "system", "content": system},
                          {"role": "user", "content": rendered_prompt}],
                response_format=NarrativeOutput,  # Pydantic model: text: str
                max_tokens=field.max_tokens or 200,
            )
            text = resp.choices[0].message.parsed.text.strip()
            if text:
                return text
        except Exception:
            if attempt == max_retries - 1:
                raise
            await asyncio.sleep(2 ** attempt)
```

`_render_prompt` scans for `{{token}}` patterns. If the token is a plain field name → context
lookup. If it contains `(`, `[`, or operators → `evaluate_expression`. If it starts with
`intake.` → `intake_data` lookup.

Apply `@traceable(run_type="llm", name="narrative_llm")` from langsmith.

### Step 13 — `orchestrator.py` [Medium]

```python
class ReportOrchestrator:
    def __init__(self, template: TemplateSchema, intake_data: dict, report_id: str):
        self.template = template
        self.intake_data = intake_data
        self.report_id = report_id
        self.context: dict[str, Any] = {}
        self._client = None  # AsyncOpenAI, lazily created

    async def generate(self) -> dict[str, Any]:
        for section in sorted(self.template.sections, key=lambda s: s.order):
            for field in section.content:
                await self._process_field(field)
        return self.context

    async def _process_field(self, field: FieldSchema) -> None:
        # 1. Evaluate condition — skip if false
        if field.condition and not self._eval_condition(field.condition):
            self.context[field.id] = field.null_value
            return
        # 2. Route to strategy
        result = await self._dispatch(field)
        # 3. Apply null_value fallback
        if result is None and field.null_value is not None:
            result = field.null_value
        self.context[field.id] = result
        # 4. Shadow augmented table back into context under the source name
        if field.strategy == "extractor" and field.source and field.computed_columns:
            self.context[field.source] = result
```

**Strategy dispatch map inside `_dispatch`:**

| strategy | handler call |
|---|---|
| `lookup` | `execute_lookup(field.source, template.lookup_sources, _lookup_ctx())` |
| `extractor` (source) | `execute_extractor_table(field.source, intake_data, computed_columns, sort_by, evaluate_expression_row)` |
| `extractor` (fields) | `execute_extractor_fields(field.fields, intake_data)` |
| `calculator` | `execute_calculator(field, context, intake_data, lookup_sources, sla_thresholds)` |
| `template_fill` | `execute_template_fill(field.template, context, intake_data, report_id, template.template)` |
| `classifier` | `execute_classifier(field, context, intake_data, lookup_sources, sla_thresholds, evaluate_expression, _get_client())` |
| `narrative_llm` | `await execute_narrative_llm(field, context, intake_data, lookup_sources, _get_client(), settings.narrative_model)` |
| `direct_input` | `intake_data.get(field.id)` |

`_lookup_ctx()` merges: `intake_data` + `context` + `{"report_id": ..., "template": {"version": ...}}`.

`evaluate_expression_row(expr, row)` is a partial of `evaluate_expression` with a single row dict
as context — used by `execute_extractor_table` for computed_columns.

`_get_client()` lazily creates `AsyncOpenAI(api_key=settings.openrouter_api_key, base_url="https://openrouter.ai/api/v1")`.
