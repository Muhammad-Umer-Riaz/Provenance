# Module 5: Review UI ‚Äî Implementation Plan

## Context

Module 4 left the system at a state where generation completes and the report status
transitions to `review`, then auto-redirects the user to `/reports/:id/review` (a stub
route that doesn't exist yet). Module 5 builds that route ‚Äî the human-in-the-loop layer
that makes field-level orchestration visible and auditable. An engineer inspects each
generated field, understands which strategy produced it, edits where needed, approves,
and unlocks export.

**Decisions confirmed:**
- `original_value` stored as a new DB column (not in metadata, not queried from audit_log)
- Right pane triggered by **click** (not hover)
- Export gate: all **non-failed** fields must be `approved`
- Edit flow: save ‚Üí `edited`, then explicitly **Approve** ‚Üí `approved` (two steps)

## Key Decisions Made During Build

- `ast.literal_eval()` used to reconstruct context from stored field values (handles Python repr for tables)
- `original_value` reset to new generated value on each regeneration (not cumulative history)
- Unapprove sets status back to `draft`; no audit log entry (no value change)
- Table field detection: `value.trim().startsWith('[')` ‚Äî can parse via `JSON.parse`
- Table fields show "cannot edit inline" note in right pane; regenerate still available
- Export button navigates to `/reports/:id/export` (Module 6 stub)
